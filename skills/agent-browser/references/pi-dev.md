pi.dev/ is the marketing landing page, not the package search UI, and browser automation may not navigate cleanly from there.
For package discovery, open https://pi.dev/packages directly and use the search box on that page.
Search with broad terms close to the user’s intent, not just exact package names.
Package names alone are noisy and produce false positives, so verify matches from the package detail page.
The package detail page inlines the README, which is the fastest reliable way to confirm commands, triggers, and behavior.
